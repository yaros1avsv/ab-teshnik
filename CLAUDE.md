# A/B Тест Помощник

Проект для планирования и запуска A/B-тестов через Claude Code с интеграцией PostHog.

## Субагенты

Claude автоматически делегирует задачи нужному субагенту по описанию. Можно также попросить напрямую: «запусти бриф» или «найди селектор».

| Субагент | Модель | Что делает |
|----------|--------|-----------|
| ab-brief | sonnet | Бриф из 10 шагов + расчёт выборки и сроков |
| ab-selector | haiku | Поиск CSS-селектора через DevTools MCP + GTM-скрипт |
| ab-experiment | sonnet | Создание feature flag + эксперимента в PostHog |
| ab-results | sonnet | Метрики и результаты экспериментов из PostHog |

Каждый субагент подгружает свой skill с task prompt при старте.

## Типичный флоу

1. «Давай спланируем A/B тест» → ab-brief собирает бриф, считает выборку, сохраняет JSON
2. «Найди элемент на странице» → ab-selector находит CSS-селектор, генерирует GTM-скрипт
3. «Создай эксперимент в PostHog» → ab-experiment создаёт flag + experiment
4. Вставь скрипт на сайт (GTM или напрямую)
5. «Покажи результаты» → ab-results получает метрики, считает significance

## Структура

- `briefs/` — JSON-файлы брифов (создаются автоматически)
- `scripts/` — GTM-скрипты для A/B тестов (создаются автоматически)
- `ab_calculator.py` — калькулятор выборки, сроков и significance (z-тест)
- `.claude/agents/` — субагенты (модель, tools, system prompt, skills)
- `.claude/skills/` — task prompts (инжектятся в субагентов)

## MCP-серверы

- **PostHog** — API для экспериментов, feature flags, метрик
- **DevTools** — Puppeteer для поиска CSS-селекторов на страницах
